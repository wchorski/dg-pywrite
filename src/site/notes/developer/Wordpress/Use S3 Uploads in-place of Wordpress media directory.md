> The wordpress file size limits still apply to the upload (default is a puny `2M`). Assuming that you're going through all the trouble to integrate a better storage solution you'll probably want to bump up this storage.

`wp-config.php`

Add this to the end. You'll see the last line referenced for context.

```php
/** S3-Uploader plugin */
define( 'S3_UPLOADS_BUCKET_URL', 'https://s3.YOURDOMAIN.site/YOUR_BUCKET_NAME' );
define( 'S3_UPLOADS_BUCKET', 'YOUR_BUCKET_NAME' );
define( 'S3_UPLOADS_REGION', 'us-east-2' ); // the s3 bucket region (excluding the rest of the URL)

// You can set access key and secret directly:
define( 'S3_UPLOADS_KEY', 'YOUR_USER_NAME' );
define( 'S3_UPLOADS_SECRET', 'SUPER_SCRET_PASSWORD' );

// Or if using IAM instance profiles, you can use the instance's credentials:
// define( 'S3_UPLOADS_USE_INSTANCE_PROFILE', true );
require_once __DIR__ . '/wp-content/plugins/vendor/autoload.php';
/** Sets up WordPress vars and included files. */
require_once ABSPATH . 'wp-settings.php';

```

`s3-endpoint.php`

> [!Warning] CORS
> Make sure you're on the same protocol as the wp site (i.e. http vs https) or you will have problems

```php
<?php
// Filter S3 Uploads params.
add_filter( 's3_uploads_s3_client_params', function ( $params ) {
	$params['endpoint'] = 'https://s3.MYDOMAIN.site:9000';
	$params['use_path_style_endpoint'] = true;
	$params['debug'] = false; // Set to true if uploads are failing.
	return $params;
} );
```

## Upload existing media from local dir to bucket

```shell
## in wp container dir
docker compose restart

docker exec -it wp-williamusic-wordpress /bin/bash
#root@b082e6c9c8f4:/var/www/html# 
wp --allow-root s3-uploads upload-directory wp-content/uploads/ uploads
```

## Fix URLs
Idk if i needed this, but it may help you if you missed a step and converted to the wrong domain

```shell
docker exec -it wp-williamusic-wordpress /bin/bash
# root@b082e6c9c8f4:/var/www/html# 
wp search-replace \
'https://MY_STORAGE_BUCKET.s3.amazonaws.com' \
'https://s3.MYDOMAIN.site:9000/MY_STORAGE_BUCKET' \
--all-tables \
--allow-root
```
